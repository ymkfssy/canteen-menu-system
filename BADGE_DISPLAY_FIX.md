# 菜品标签显示修复说明

## 修改内容

将菜品标签（畅销、推荐）从浮动显示改为紧跟在菜品名称后面显示。

## 修改的文件

### 1. 前台显示样式 (`public/css/style.css`)
- 移除了 `.dish-badges` 的绝对定位
- 改为 `inline-flex` 布局，直接跟在名称后面
- 调整标签大小和间距，使显示更紧凑

### 2. 前台显示逻辑 (`public/js/display.js`)
- 修改DOM结构，将标签和菜品名称放在同一个容器内
- 使用flex布局让名称和标签在同一行显示
- 标签容器紧跟在菜品名称后面

### 3. 后台管理样式 (`public/admin/css/admin.css`)
- 同样修改 `.dish-badges` 样式为行内显示
- 修改 `.dish-name-section` 为横向布局
- 调整标签大小，保持与前台一致

### 4. 后台管理逻辑 (`public/admin/js/admin.js`)
- 将菜品名称从 `div` 改为 `span`，方便行内显示
- 保持标签HTML结构不变，只调整显示方式

## 显示效果

### 修改前：
- 标签浮动在菜品项右上角
- 可能与菜品名称重叠或挤占空间
- 标签使用垂直布局

### 修改后：
- 标签直接跟在菜品名称后面
- 不挤占菜品名称空间
- 标签使用水平布局，更紧凑
- 示例：`红烧肉 [畅销][推荐] ¥18`

## 样式特点

- **标签大小**：更小更紧凑（9px字体，1px内边距）
- **圆角**：从12px改为8px，更精致
- **间距**：标签之间4px间距，与名称6px间距
- **换行控制**：`white-space: nowrap` 防止标签换行

## 兼容性

- 前台展示页面和后台管理页面统一显示方式
- 保持原有的动画效果（fadeIn 0.3s）
- 标签颜色和样式保持不变

现在菜品标签会以更简洁的方式直接跟在菜品名称后面，不会影响页面布局和菜品名称的显示。
